ℹ️ Acerca de este proyecto

Documentación sobre la metodología, las fuentes de datos y la estructura del catálogo de Open Data Canarias.

🔬 Metodología de recopilación

Los datos se recopilan manualmente a partir de fuentes oficiales: portales de transparencia, portales CKAN, sedes electrónicas y páginas institucionales de cada entidad. Cada entrada se valida contra el schema JSON oficial del proyecto antes de incorporarse al catálogo.

El campo verification.status indica el nivel de confianza de cada registro:

  • verified — dato comprobado directamente en fuente oficial.
  • estimated — estimado a partir de indicios fiables.
  • inferred — deducido de otras fuentes.
  • pending — pendiente de verificación.

🧩 Campos principales del catálogo

Campo Descripción Valores posibles
id Identificador único de la entidad Texto en minúsculas con guiones bajos
name Nombre oficial de la entidad Texto libre
entity_kind Tipo de entidad gobierno_autonomico · cabildo · ayuntamiento · organismo · empresa_publica
scope Ámbito administrativo autonomico · insular · municipal · estatal · europeo
islands Islas a las que pertenece la entidad Tenerife · Gran Canaria · Lanzarote · Fuerteventura · La Palma · La Gomera · El Hierro · La Graciosa · Todas
portals Lista de portales publicados por la entidad Array de objetos (ver schema)
portals.kind Tipo de portal open_data · transparencia · estadistica · geoespacial · agregador · other
portals.machine_readable ¿Los datos son legibles por máquina? true / false
portals.has_api ¿Dispone de API? true / false / null
verification.status Nivel de confianza del registro verified · estimated · inferred · pending

🗓️ Actualización del catálogo

El catálogo se actualiza manualmente. La fecha de última revisión de cada portal se refleja en el campo portals.last_updated (formato YYYY-MM). El script src/static/data/fecha.py automatiza la consulta a APIs CKAN para mantener esas fechas al día.
